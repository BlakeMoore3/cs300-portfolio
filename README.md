CS 300 Portfolio Reflection

What was the problem you were solving in the projects for this course?

In this course, I developed a program that reads course data from a file and allows the user to search and print course information. The main problem was figuring out the most efficient data structure to store and retrieve the information.

How did you approach the problem?

I approached this by looking at different data structures. Vector, hash table, and binary search tree. I needed to understand how each structure handled insertion, searching, and sorting. Data structures matter because they directly affect runtime performance and memory.

How did you overcome any roadblocks you encountered while going through the activities or project?

I struggled at times with pointer logic in hash tables and tree traversal. I overcame these roadblocks by breaking problems into smaller pieces and testing logic step by step.

How has your work on this project expanded your approach to designing software and developing programs?

This helped me think more about the effiency of code before writing it. Instead of just making something work, I now consider the time complexity and scalability.

How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?

I focused on structuring my code clearly with functions like LoadCourse and PrintCourse. Separating things made the program easier to debug and expand.
